# 青禾邑无界面自动试玩

## 框架

tests/headless_playtest.gd 直接驱动正式 GameState，不会复制经济或战斗公式，也不创建窗口。自动玩家会处理事件，并根据每日账簿、来敌时间和战前推演执行建造、交易、征兵、巡剿、政令、城池晋升与时代更迭。

阵令也走正式入口：均衡路线在军力或民心承压时使用固守阵，安全后才按远射兵比例切换远射阵；重农固定固守，重军采用无修正军阵。专项 `tests/order_balance.gd` 另以 3 种编成 × 4 种阵令、每格 3000 次战斗验证收益与代价。

默认包含四种行为路线：

- **均衡经营**：先建立基础城防，再按敌军梯队扩军，同时维持各类生产。
- **重农发展**：优先升级农田、林场、石场和市场，敌军临近时才集中备战。
- **重军备战**：先满足当前梯队的基础军力，再追求额外城防与高士气，资源余量更低。
- **贪发展不设防**：只建设经济，不主动扩军或修墙，作为防务价值的对照组。

另有“每日巡剿”机制滥用探针，专门检查能否通过连续巡剿永久延迟攻城、打空敌军后继续领奖。

框架每 15 日执行一次存档往返，并持续检查：

- 资源有限、非负且不超过仓储容量。
- 民口、军籍和伤员非负，总居民不超过住房容量。
- 可战士卒与伤员共同受军籍容量约束。
- 伤员人数与康复队列逐兵种一致。
- 建筑、民心和敌军编成不越界。
- 已建建筑不超过当前城池用地；时代、时代积累和城池等级在存档往返后完全一致。
- 征兵只转移人口，不凭空制造居民。

测试时只关闭自动存档和诊断写盘；正常游戏默认仍开启，两者的正式行为没有改变。

## 运行

~~~bash
HOME="$PWD/.home" ./tools/godot/Godot.app/Contents/MacOS/Godot \
  --headless --path . --script tests/headless_playtest.gd -- \
  --runs=1000 --days=60 --seed=20260715
~~~

可用参数：

- --runs=N：每种策略运行局数，默认 1000。
- --days=N：模拟终点，默认第 60 日。
- --seed=N：基础随机种子。
- --policy=balanced|agrarian|militarist|greedy：只运行一种策略。
- --report=res://...json：指定 JSON 报告路径；同时生成同名 Markdown。
- --strict：把平衡警告也视作失败；状态一致性错误始终失败。

## 结论与观察

- 防务有决定性回报：不设防路线虽然囤积大量粮财和人口，但 60 日全部守城胜率只有 8.0%，终局稳定率为 0%。
- 十事件与阵令版本的三种正常路线在 60 日稳定率为 74.3%—81.3%，180 日为 98.8%—100%；重农用坚壁显著减少伤亡，重军用锋矢换取最高首战胜率与更高伤亡，均衡会随弓手占比在持重和雁行之间调整。
- 敌军会随胜利推进，因此 60 日全程胜率最终会靠近动态难度平衡点；不能只用全程胜率判断重军路线是否有效，首战、推进梯队、终局稳定和资源余量需要合看。
- 正常路线的粮与财日净额没有出现负数。当前经济压力主要来自建造、征募、巡剿、政令和战败损失，而非被动维持费把库存持续拖到零；这是后续真人试玩需要重点感受的部分。
- 60 日内三种正常路线都几乎必然经历至少一次战败。现版本强调“受挫后恢复”而非无损通关；若真人感受过于挫败，应优先调整长期梯队节奏，而不是继续提高初始资源。

## 本轮发现并修复

1. 伤员未占军籍容量，征兵后康复会造成超编。现改为可战士卒与伤员共同占用容量。
2. 战败仍自动升级下一波敌军，形成伤亡、低士气与更强敌军叠加的死循环。现改为胜利才推进梯队，失败重遇当前梯队并获得额外两日整顿。
3. 巡剿可每日重复延迟、打空敌军后继续领奖。现改为每支敌军最多延误一日，城外歼灭会立即推进下一梯队，巡剿不再净赚财货。
4. 满级城防与军籍存在硬上限，但敌军原先无限增长，长局最终必然进入不可恢复连败。现改为八档主力上限，后续保留挑战与编成变化但不再越过玩家成长上限。
5. 每次胜利都立即提升敌军档位，使重军路线因推进更快反而更早受罚。现改为前三档逐次提升、此后每档持续两次胜利；自动诊断同时联合首战、全程胜率和终局稳定性判断军备回报。
